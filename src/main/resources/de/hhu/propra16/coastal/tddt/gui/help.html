Benutzerhandbuch

1 Menü

  • Datei

    • Nach Programmstart können Sie zunächst eine Aufgabe auswählen. Dazu gehen Sie auf Datei → Öffne Katalog, suchen dort nach ihrem XML-Dokument und öffnen dieses anschließend. Rechts sollten Sie nun einen Katalog sehen, aus dem Sie eine Aufgabe auswählen können.
    Beim Start einer Aufgabe erscheint ggf. ein Fenster mit der Frage, ob Sie an ihrer zuletzt gespeicherten Version weiterarbeiten möchten. Oben links wird eine kurze Beschreibung des Arbeitsauftrags angezeigt.
    Beim Verlassen einer Aufgabe werden Sie gefragt, ob Sie Ihren Fortschritt der momentanen Aufgabe speichern möchten, sodass bei erneutem Starten dieser Aufgabe die Textfelder mit ihrem gespeichertem Inhalt aufgerufen werden können.

  • Speichern

    • Wenn Sie Speichern auswählen, wird der Fortschritt der momentanen Aufgabe wie oben erläutert gespeichert.

  • Programm schließen

    • Schließt das Programmfenster.


    2 Arbeitsstatus

    Es gibt vier Status: RED, GREEN, REFACTOR CODE und REFACTOR TEST.

  • RED

    • In diesem Status müssen Sie in das obere rechte Textfeld einen Test zur Aufgabe schreiben, der fehlschlägt.
    Wenn Sie auf den Button Nächster Schritt drücken, werden Ihnen im unteren rechten Textfeld entsprechende Meldungen (z. B. Kompilierfehler) angezeigt.
    Sobald es keine Fehler bei den zu schreibenden Tests mehr gibt, sondern nur noch beim zu bearbeitenden Programm, wechselt der Status zu GREEN.

  • GREEN

    • Um in den nächsten Status überzugehen, müssen Sie den Programmcode im oberen linken Textfeld ergänzen bzw. verändern, bis alle Tests im oberen rechten Textfeld erfüllt sind. Beim Klick auf Nächster Schritt gelangen Sie, sofern das untere linke Textfeld keine Meldungen anzeigt, in den Status REFACTOR CODE.
    Sollten Sie einen Fehler beim Schreiben der Tests im Status RED gemacht haben, können Sie auf Schritt zurück drücken, jedoch würden dabei Ihr Code und Ihre Tests zu dem Zeitpunkt zurückgesetzt, zu dem Sie zuletzt am Anfang des Status RED gewesen sind.

  • REFACTOR CODE/TEST

    • In diesen beiden Status können Sie u. a. Einrückungen und Kommentare vornehmen, Variablennamen ändern sowie die Wartbarkeit und Erweiterbarkeit Ihres Codes/Ihrer Tests verbessern.
    Mit einem Klick auf Nächster Schritt kommen Sie von REFACTOR CODE zu REFACTOR TEST bzw. von REFACTOR TEST zu RED.


    3 Erweiterungen

  • Babysteps

    • Wenn für eine Übung Babysteps eingeschaltet sind, wird die Zeit, die Sie in den Phasen RED und GREEN haben, limitiert.
    Die verbleibende Zeit wird rechts neben dem Schriftzug Verbleibende Zeit angezeigt und im Sekundentakt heruntergeschaltet.
    Ist die Zeit abgelaufen, wird der neue Test/Code gelöscht und es wird in die vorangegangene Phase zurückgewechselt.

  • Tracking

    • Ist diese Funktion aktiviert, werden die Zeiten, die Sie in den einzelnen Phasen verbringen, aufgezeichnet.
    Unter Statistiken → Benutzeranalyse können Sie die Zeiten in Form eines Kuchendiagrammes einsehen.
